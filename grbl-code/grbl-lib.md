### 一、Grbl 核心定义（一句话讲明白）
Grbl 是一款**开源、高性能、轻量级的G代码解析与CNC运动控制固件**，专门为AVR单片机（比如Arduino Nano/Uno的ATmega328P）设计，能把标准G代码转换成步进电机的精准运动控制信号，是低成本CNC设备（雕刻机、激光机、小型铣床，也包括你的拼豆机）的“大脑”。

### 二、Grbl 核心特点（为什么适合你的项目）
#### 1. 硬件适配性：完美匹配你的设备
- **运行硬件**：仅需Arduino Nano/Uno（ATmega328P 16MHz），和你当前的开发板完全一致；
- **外设适配**：原生支持CNC Shield（X/Y/Z轴步进电机驱动），你的双42步进电机（NEMA17）可以直接对接，无需额外改硬件；
- **串口通信**：默认115200波特率（和你代码的波特率一致），通过串口发送G代码即可控制电机运动。

#### 2. 性能优势：精准、高效、稳定
- **高频脉冲输出**：能稳定输出30kHz的步进脉冲，保证电机运动无抖动、精度高（NEMA17 200步/圈的电机，30kHz意味着最高转速可达900RPM，远超拼豆机需求）；
- **前瞻规划（Look Ahead）**：能提前解析18个运动指令，自动规划加速度和速度，实现“平滑加减速+无顿挫转弯”，避免电机卡顿；
- **全加速度管理**：支持独立的轴加速度/速度设置，适配不同电机的性能差异。

#### 3. 功能完整性（覆盖CNC核心需求）
Grbl支持几乎所有你做拼豆机需要的G代码和功能：
| 核心功能                | 对应G代码/指令                | 对你的价值                          |
|-------------------------|-------------------------------|-------------------------------------|
| 直线运动                | G0（快速定位）、G1（进给运动） | 控制X/Y轴步进电机精准移动到指定位置 |
| 圆弧/圆形运动           | G2（顺时针）、G3（逆时针）| 若拼豆机需要圆弧轨迹，可直接使用    |
| 单位/距离模式           | G20（英寸）、G21（毫米）、G90（绝对坐标）/G91（相对坐标） | 灵活定义运动坐标体系                |
| 主轴/舵机控制           | M3/M4/M5（主轴启停）、S值（转速） | 可复用主轴控制逻辑驱动你的MG90S舵机 |
| 限位/归位               | G28（回零）、硬件限位引脚支持  | 实现拼豆机“原点校准”，避免超程      |
| 实时状态反馈            | 串口实时上报电机位置/速度      | 替代你当前的“200ms状态打印”，更高效 |

#### 4. 轻量级：适配Nano的内存
- Grbl v1.1精简版仅占用ATmega328P约80%的Flash（32KB Flash用25KB左右）、约60%的SRAM（2KB用1.2KB左右）；

⚠️ **关键注意事项（适配你的硬件）**：
- v0.9及以上版本默认波特率是115200（和你代码一致），v0.8及以下是9600；
- v0.9后Z限位引脚（D12）和主轴使能引脚（D11）互换：你的舵机接了D12，需要在Grbl源码中重新映射引脚，避免冲突；
- Grbl默认控制X/Y/Z三轴，你只需用到X/Y轴，可注释掉Z轴相关代码进一步省内存。

### 三、落地步骤：Grbl集成到你的项目（新手友好）
#### 步骤1：下载适配的Grbl版本（选v0.9j，适配Nano+你的引脚）
1. 下载Grbl v0.9j源码：https://github.com/gnea/grbl/archive/refs/tags/v0.9j.zip
2. 解压后，找到`grbl/grbl_config.h`（引脚配置文件），修改为你的硬件引脚（适配CNC Shield）：
   ```cpp
   // 替换grbl_config.h中的默认引脚定义，匹配你的CNC Shield
   #define X_STEP_PIN     2  // 你的电机1 STEP
   #define X_DIR_PIN      5  // 你的电机1 DIR
   #define Y_STEP_PIN     3  // 你的电机2 STEP
   #define Y_DIR_PIN      6  // 你的电机2 DIR
   #define STEPPER_ENABLE_PIN 8  // 你的使能引脚
   // 注释掉Z轴（不用），节省内存
   // #define Z_STEP_PIN     4
   // #define Z_DIR_PIN      7
   // 调整舵机/主轴引脚（避免冲突）
   #define SPINDLE_ENABLE_PIN 10  // 不用D12（你的舵机引脚），改到D10
   ```

#### 步骤2：编译上传Grbl固件到Nano
1. 打开Arduino IDE，导入Grbl源码（选择`grbl/grbl.ino`）；
2. 开发板选择：`Arduino Nano → ATmega328P (Old Bootloader)`；
3. 串口波特率默认115200（和你代码一致），点击上传；
4. 上传成功后，打开串口监视器，发送`$$`，会返回Grbl的配置参数（说明固件正常运行）。

#### 步骤3：保留舵机控制逻辑（Grbl扩展）
Grbl支持自定义M代码（非标准M代码），你可以在Grbl源码中新增舵机控制指令（比如M100=舵机转到指定角度）：
1. 找到`grbl/gcode.c`中的`gcode_execute_m_code()`函数；
2. 新增M100指令解析：
   ```cpp
   // 在gcode_execute_m_code()中添加
   case 100: // M100 Sxx → 舵机转到xx度
     if (parser.seen('S')) {
       int servo_angle = parser.value_int();
       servo_angle = constrain(servo_angle, 0, 180);
       myServo.write(servo_angle); // 你的舵机控制逻辑
     }
     break;
   ```
3. 重新编译上传，即可通过发送`M100 S90`控制舵机转到90度。

#### 步骤4：实现互斥运动（Grbl配置）
Grbl默认支持多轴联动，你只需通过配置限制“同一时间仅单轴运动”：
1. 发送Grbl配置指令：`$110=300`（X轴最大速度300步/秒）、`$111=300`（Y轴最大速度300步/秒）；
2. 发送G代码时，仅发送单轴指令（如`G1 X200 F300`→仅X轴运动，`G1 Y200 F300`→仅Y轴运动），即可实现互斥。

#### 步骤5：测试运动效果
1. 串口发送`G90`（绝对坐标模式）；
2. 发送`G1 X200 F300`→电机1正转200步（拼豆机单次运动步数）；
3. 发送`G1 X0 F300`→电机1反转回原点；
4. 发送`M100 S90`→舵机转到90度；
   ✅ 此时电机运动平滑无卡顿，舵机控制正常，互斥逻辑通过单轴G代码实现。

### 四、避坑提醒（新手常踩）

2. **内存优化**：注释掉Grbl中不用的功能（如Z轴、圆弧运动、冷却液控制），可节省约200字节RAM，适配Nano的2KB内存；
3. **波特率**：Grbl v0.9+默认115200，和你代码一致，无需修改；
4. **使能引脚**：Grbl的使能引脚是低电平有效，和你的CNC Shield一致，无需改逻辑。
